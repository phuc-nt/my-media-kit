## Session 2 — MLX-first pivot

### Shipped

- **ADR-011 (Tauri pivot)** + **ADR-012 (MLX-first on Apple Silicon)** added to `docs/architecture-decisions.md`. Cloud providers moved out of the MVP critical path; local MLX becomes the primary default on Apple Silicon. `docs/llm-tasks.md` updated with concrete model names per feature + "default per platform" matrix.
- `transcription-kit::MlxWhisperTranscriber` — spawns `mlx_whisper` CLI, parses `--output-format json` into `TranscriptionSegment`s with word timestamps + language tag. Gated `cfg(all(target_os = "macos", target_arch = "aarch64"))`. Default model: `mlx-community/whisper-large-v3-turbo`. [src-tauri/crates/transcription-kit/src/mlx_whisper.rs](../../src-tauri/crates/transcription-kit/src/mlx_whisper.rs)
- `ai_kit::MlxLmProvider` — wraps `OpenAiProvider` at `127.0.0.1:8080`, implements `Provider` trait with `AiProviderType::Mlx`. Downgrades structured-output requests to freeform (mlx_lm.server doesn't honour `response_format.json_schema`), appends an explicit JSON contract to the prompt, then extracts the first balanced JSON value from the response via an in-file extractor that handles fenced blocks, prose preambles, array roots, and brace-in-string edge cases. 6 extractor unit tests. [src-tauri/crates/ai-kit/src/providers/mlx_lm.rs](../../src-tauri/crates/ai-kit/src/providers/mlx_lm.rs)
- Tauri commands wired: `mlx_whisper_transcribe`, `content_filler_detect`, `content_summary`, `content_chapters`. Provider resolution registers MLX first on Apple Silicon; cloud providers still available when a keyring entry exists. [src-tauri/src/commands/transcription.rs](../../src-tauri/src/commands/transcription.rs) + [src-tauri/src/commands/content.rs](../../src-tauri/src/commands/content.rs)

### Fixed

- **URL double-`/v1` bug** in `MlxLmProvider::new`: the embedded `OpenAiProvider` appends `/v1/chat/completions` internally, so we must store the bare server host. Previously produced `404 Not Found` against `/v1/v1/chat/completions`. Unit test `host_strips_trailing_v1_and_slash` pins the behaviour.
- **JSON extractor array-root bug**: initial implementation tried `{` first, matching a nested object inside an array and returning early. Fixed by picking the earliest opener offset in the text.

### Known follow-ups (after session 2)

- Filler detection quality on Qwen 3B: false positives on "Today/First/Then/Finally". Either prompt tune or upgrade default to Qwen 7B for filler.
- Auto-spawn `mlx_lm.server` from the app (currently manual). Adds lifecycle + port-busy handling complexity.

- MLX whisper sidecar writes temp files; should clean up on abort signal.
- Installer story for macOS: user must `pip install mlx-lm mlx-whisper` manually; investigate bundling a `uv` venv.
