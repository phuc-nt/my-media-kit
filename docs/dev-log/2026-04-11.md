## Session 2 — MLX-first pivot

### Shipped

- **ADR-011 (Tauri pivot)** + **ADR-012 (MLX-first on Apple Silicon)** added to `docs/architecture-decisions.md`. Cloud providers moved out of the MVP critical path; local MLX becomes the primary default on Apple Silicon. `docs/llm-tasks.md` updated with concrete model names per feature + "default per platform" matrix.
- `transcription-kit::MlxWhisperTranscriber` — spawns `mlx_whisper` CLI, parses `--output-format json` into `TranscriptionSegment`s with word timestamps + language tag. Gated `cfg(all(target_os = "macos", target_arch = "aarch64"))`. Default model: `mlx-community/whisper-large-v3-turbo`. [src-tauri/crates/transcription-kit/src/mlx_whisper.rs](../../src-tauri/crates/transcription-kit/src/mlx_whisper.rs)
- `ai_kit::MlxLmProvider` — wraps `OpenAiProvider` at `127.0.0.1:8080`, implements `Provider` trait with `AiProviderType::Mlx`. Downgrades structured-output requests to freeform (mlx_lm.server doesn't honour `response_format.json_schema`), appends an explicit JSON contract to the prompt, then extracts the first balanced JSON value from the response via an in-file extractor that handles fenced blocks, prose preambles, array roots, and brace-in-string edge cases. 6 extractor unit tests. [src-tauri/crates/ai-kit/src/providers/mlx_lm.rs](../../src-tauri/crates/ai-kit/src/providers/mlx_lm.rs)
- Tauri commands wired: `mlx_whisper_transcribe`, `content_filler_detect`, `content_summary`, `content_chapters`. Provider resolution registers MLX first on Apple Silicon; cloud providers still available when a keyring entry exists. [src-tauri/src/commands/transcription.rs](../../src-tauri/src/commands/transcription.rs) + [src-tauri/src/commands/content.rs](../../src-tauri/src/commands/content.rs)

### Fixed

- **URL double-`/v1` bug** in `MlxLmProvider::new`: the embedded `OpenAiProvider` appends `/v1/chat/completions` internally, so we must store the bare server host. Previously produced `404 Not Found` against `/v1/v1/chat/completions`. Unit test `host_strips_trailing_v1_and_slash` pins the behaviour.
- **JSON extractor array-root bug**: initial implementation tried `{` first, matching a nested object inside an array and returning early. Fixed by picking the earliest opener offset in the text.

### Verified

- **Real `mlx_whisper` test** (`crates/transcription-kit/tests/mlx_whisper_smoke.rs`) against `~/Downloads/mytrans_term.mov`:
  - 2 segments detected, `lang=vi`, word-level timestamps + confidence present.
  - Runtime: 6.68 s (warm cache).
- **Real `mlx_lm.server` integration** with `mlx-community/Qwen2.5-3B-Instruct-4bit` on port 8080 (`crates/content-kit/tests/mlx_lm_smoke.rs`), 3/3 tests pass:
  - **Filler detect**: 5 shape-valid detections returned (quality: 1 correct "um" + 4 false positives — prompt tuning TODO).
  - **Summary (Brief)**: 2-sentence coherent English narrative of the test transcript.
  - **Chapters**: 5 entries, first pinned to `00:00`, titles match transcript topics.
  - Runtime: 9.42 s across all three.
- `cargo test --workspace` + `CREATOR_UTILS_TEST_MEDIA` + live `mlx_lm.server` → **126 passing, 0 failing** (+21 since last report).
- Related: `plans/reports/test-260411-0920-mlx-first-transcription-and-llm.md`

### Known follow-ups (after session 2)

- Filler detection quality on Qwen 3B: false positives on "Today/First/Then/Finally". Either prompt tune or upgrade default to Qwen 7B for filler.
- Auto-spawn `mlx_lm.server` from the app (currently manual). Adds lifecycle + port-busy handling complexity.
- Model switcher UX: swapping requires restarting the server. Consider pre-loading multiple models on separate ports if RAM allows.
- MLX whisper sidecar writes temp files; should clean up on abort signal.
- Installer story for macOS: user must `pip install mlx-lm mlx-whisper` manually; investigate bundling a `uv` venv.

## Session 3 — Translate feature

### Shipped

- **`content-kit::translate`** — transcript translation module. Rule v2: if source language is Vietnamese, skip translation; otherwise default target is Vietnamese. Callers can override the target via `TranslateOptions`. Parallel-array JSON schema (one `translations` string per input segment) keeps small local models honest. [src-tauri/crates/content-kit/src/translate.rs](../../src-tauri/crates/content-kit/src/translate.rs)
- Helpers: `should_skip` (primary-tag comparison, case-insensitive, collapses `vi-VN` → `vi`), `language_display_name` (BCP-47 → LLM-facing English name, 13 common tags).
- `TranscriptionSegment` timestamps + word arrays preserved 1:1; only `text` and `language` are updated after translation. Word re-alignment across languages is intentionally out of scope.
- Tauri command `content_translate` wired to the registry-aware provider resolver. Accepts `{provider, model, segments, sourceLanguage?, targetLanguage?}`. Default target is `"vi"`.

### Verified

- **Unit tests** (content-kit `translate` module): 8 new, all passing — skip logic, display name mapping, prompt shape, schema, empty input, length-mismatch error, passthrough happy path.
- **Real-clip integration** (`crates/content-kit/tests/translate_smoke.rs`) against 30-second trims pulled from `test-input/*.mp4`:
  - **VN clip** (`Su-that-ve-tam-ly-hoc-khong-gi`): whisper detects `vi` → runner skips, 0 provider calls. ✅
  - **EN clip** (`What-Makes-a-Good-Life-Lessons`): whisper detects `en`, 6 segments; MLX Qwen2.5-7B translates all 6 to Vietnamese. Sample: *"what their most important life goals were"* → *"những mục tiêu cuộc sống quan trọng nhất của họ là gì"*. ✅
  - **JP clip** (`Hope-invites-Tsutomu-Uematsu`): whisper detects `ja`, 13 segments; MLX translates to Vietnamese. Sample: *続けるって大事です* → *"Tiếp tục là rất quan trọng."* ✅
- Full runtime: 32.58 s for the 3 integration tests on Qwen2.5-7B-4bit.
- Full workspace: `cargo test --workspace` → **137 passing** (+11 since session 2).

### Changed

- `MlxLmProvider` default model bumped in docs to Qwen2.5-7B for translate / summary / chapters. 3B is still fine for filler detection.
- `content-kit/Cargo.toml` dev-deps: `transcription-kit` added so the smoke test can call whisper inline.

### Known follow-ups (after session 3)

- Length-mismatch recovery: if the LLM returns N-1 translations for N lines, we currently error out. A repair pass (re-ask for missing index) would harden local-model quality.
- VN clip returned only 1 segment for the 30 s trim because whisper auto-joined — not a bug, but makes the test less rigorous for the skip path. Consider a longer VN fixture.
- Frontend translate panel not wired yet; the command exists but no UI.

## Session 5 — Milestone A (demo end-to-end)

### Shipped

- **`docs/backlog.md`** — priorities for Milestones A / B / C plus scoring legend and open questions.
- **Shared Tauri `AppState`** ([src-tauri/src/state.rs](../../src-tauri/src/state.rs)) with two `Mutex<HashMap<PathBuf, Arc<…>>>` caches: `pcm` (raw 16 kHz samples) and `transcripts` (full whisper results). Commands use `State<'_, AppState>` to read/write; 3 unit tests for the cache round-trip + clear semantics.
- `detect_silence_in_file` refactored: hit the cached PCM when present, ffmpeg-extract + cache otherwise. Response carries `from_cache: bool` so the UI can show the fast-path indicator.
- `mlx_whisper_transcribe` refactored: hit cached transcript unless `force: true` is passed; otherwise run whisper, stash the result, return the (now cached) entry. Also added `get_cached_transcript(path)` + `clear_cache(path?)` commands for the frontend.
- **Shared frontend `source-store`** ([src/js/source-store.js](../../src/js/source-store.js)): single source of truth for the active source path + last-known transcript. Emits `change` events so every view auto-refreshes placeholders.
- **Sidebar source panel** ([src/js/source-panel.js](../../src/js/source-panel.js)) with path input, live meta caption (`basename + transcript segments count`), and a Clear-cache button that invokes `clear_cache`.
- **New feature views** wired to the store + backend:
  - `features/transcribe.js` — runs `mlx_whisper_transcribe`, renders language + `HH:MM:SS | text` table, auto-loads cached transcript when switching back to the tab.
  - `features/translate.js` — guards on `requireTranscript`, runs `content_translate`, renders a 3-column side-by-side table (time / original / translated). Surfaces the skip case with a `<strong>skipped</strong>` badge.
  - `features/summary.js` — style + language picker, renders the text in a `<pre class="summary-text">`.
  - `features/chapters.js` — runs `content_chapters`, shows the table, and has a **Copy YouTube format** button that writes `mm:ss title` lines to the clipboard.
- `features/autocut.js` refactored to pull the source path from `source-store` (no more local input) and show the `from_cache` badge after the first run.
- `util.js` helpers (`formatMs`, `setStatus`, `requireSource`, `requireTranscript`, `escapeHtml`, `renderErrorBox`) shared across feature views.
- CSS additions: `.source-picker` sidebar block, `pre.summary-text` output style, small-ghost button variant.

### Verified

- `cargo test --workspace` → **140 passing, 0 failing** (+3 new AppState tests).
- `cargo check` → clean.
- Frontend surface audited for Tauri serialization: `TranscriptionSegment` serialises snake_case (no `rename_all`), feature views read `start_ms` / `end_ms` directly; silence + cache output uses camelCase `fromCache` → frontend handles both shapes.
