## Research: Filler Word Detection Approaches

**Goal:** remove "ờ", "ừm", "à" filler sounds that survive the existing silence-based cut pipeline.

### What was tried

**`--initial-prompt` trick** — add `"ừm, ờ, à,"` to mlx_whisper's `--initial-prompt` flag to
hint the model into keeping filler tokens. **Result: no effect.** Whisper suppresses fillers at
the model/training level (trained on cleaned subtitles), not at the decoding layer. No config
flag can override this.

**CrisperWhisper** — `nyrahealth/CrisperWhisper`, a Whisper fine-tune with filler tokens added
to the tokenizer/vocabulary. Converts "um"/"uh" → explicit tokens with precise timestamps.
**Result: dead end for Vietnamese.**
- Model is English-only (fine-tuned on English corpus).
- `faster-whisper` (CTranslate2) needed; `mlx_whisper` can't load it directly.
- After CT2 conversion + loading: `ValueError: Invalid input features shape: expected (1,128,3000)
  got (1,80,3000)` — model expects 128 mel bins (v3-style) but output was 80. Shape mismatch
  unresolved.
- Even if fixed: Vietnamese fillers differ acoustically from English "um"/"uh"; model untrained
  on them.

**Silero-VAD** — frame-level speech probability model (32ms frames, p∈[0,1]) from snakers4.
**Result: promising — detects "ờ" filler blob pattern.**

### Silero-VAD Findings

Ran VAD on the autocut output video across 5 long word spans (≥ 600ms in output transcript):

| Word | Span (output) | Pattern found |
|------|---------------|---------------|
| tôi  | 680..1720ms   | [speech 704..1024ms] [non-sp 1056..1152ms] **[speech-blob 1184..1344ms = ờ!]** [non-sp 1376..1696ms] |
| cuốn | 2800..3760ms  | trailing silence 3488..3648ms (160ms) |
| đọc  | 4240..5520ms  | interior silence 4704..5088ms (384ms) |
| Cốt  | 6360..7440ms  | interior silence 6624..7264ms (640ms) |
| của  | 12500..14240ms | trailing silence 12672..13792ms (1120ms) |

**Key insight:** "ờ" shows up as a short speech blob (160ms) sandwiched between two non-speech
regions inside the "tôi" word span. Whisper absorbed the "ờ" time into "tôi"'s timestamp.
Pattern: `[word-speech][gap][filler-blob(≤350ms)][gap]` → cut the whole block.

### POC Algorithm (Python)

```python
def find_vad_cuts_v2(word_start, word_end, min_cut_ms=100, edge_buffer_ms=60, max_filler_blob_ms=350):
    # Build runs of (is_speech, start_ms, end_ms)
    runs = get_runs(word_start, word_end)
    cuts = []
    for i, r in enumerate(runs):
        if r['speech']: continue
        prev, nxt = runs[i-1] if i>0 else None, runs[i+1] if i+1<len(runs) else None
        after = runs[i+2] if i+2<len(runs) else None
        # Filler-blob pattern: [speech][non-sp][short-speech][non-sp]
        if nxt and nxt['speech'] and after and not after['speech']:
            blob_dur = nxt['e'] - nxt['s']
            if blob_dur <= max_filler_blob_ms:
                cuts.append((r['s'], after['e'], "filler-blob"))
                continue
        # Plain interior / trailing non-speech
        if (r['e'] - r['s']) >= min_cut_ms:
            cuts.append((r['s'], r['e'], "trailing/interior"))
    return cuts
```
